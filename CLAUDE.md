# Noa Swimming - Competitive Swimming Performance Data Platform

This is Noa's competitive swimming data platform, built by his father Jaco. Noa is a competitive swimmer in Alberta, Canada. The platform collects, processes, and analyzes swim meet results to track performance and development.

## Architecture

- **Language/tooling:** Python project managed with `uv`
- **Backend:** Supabase (PostgreSQL + API)
- **Pipeline:** Scrapers → Processors → Database (modular, each stage independent)

### Directory Structure

```
src/
  scrapers/        # Data collection from external sources (Hy-Tek web results, Swimcloud, etc.)
  processors/      # Data transformation: time parsing, swimmer deduplication, course conversions
  database/        # Supabase client, schema helpers, query utilities
  runner.py        # CLI entry point for running pipeline stages
  config.py        # Environment and configuration management
agents/            # Claude Code agent instruction files for autonomous operation
scripts/           # Utility scripts (one-off migrations, data fixes)
tests/             # Test suite
docs/              # Documentation
```

## Swimming Domain Knowledge

### Course Types

| Code | Name | Pool Length | Notes |
|------|------|------------|-------|
| SCY | Short Course Yards | 25 yards | Primarily used in the USA |
| SCM | Short Course Meters | 25 meters | Used internationally for short course |
| LCM | Long Course Meters | 50 meters | Olympic standard, used at Worlds/Olympics |

In Alberta and Canada, most club meets are SCM. LCM is used for major competitions (Nationals, Trials). SCY is rarely used in Canada except at cross-border meets.

### Standard Events

**Freestyle:** 50, 100, 200, 400, 800, 1500
**Backstroke:** 50, 100, 200
**Breaststroke:** 50, 100, 200
**Butterfly:** 50, 100, 200
**Individual Medley (IM):** 200, 400

Not all events are offered at all meets or in all age groups. 50s of stroke are often senior-only. 800 Free is traditionally a women's event and 1500 Free a men's event at major competitions, though this has changed at the Olympic level.

### Time Format

Times are formatted as `MM:SS.ss` or `SS.ss`:
- `1:02.34` = 1 minute, 2.34 seconds = 62.34 total seconds
- `28.91` = 28.91 seconds
- Splits may appear as cumulative or interval times

When converting or comparing times, always work in total seconds internally.

### Age Groups

| Age Group | Notes |
|-----------|-------|
| 10 & Under | Youngest competitive group |
| 11-12 | |
| 13-14 | |
| 15-16 | |
| 17-18 | |
| Senior/Open | No age restriction |

Age is typically determined as of December 31 of the competition year (for Canadian swimming). Age group standards and qualifying times vary by province and organization.

### Key Organizations

- **World Aquatics** (formerly FINA): International governing body, sets world records and competition rules
- **Swimming Canada:** National governing body
- **Swim Alberta:** Provincial section, runs provincial championships and maintains Alberta records
- **USA Swimming:** US governing body (relevant for cross-border comparisons and SCY data)

### Meet Types

- **Invitational:** Club-hosted, open entry, good for racing experience
- **Provincial (Provincials):** Swim Alberta championships, require qualifying times
- **National (Nationals/Trials):** Swimming Canada championships, require national qualifying times
- **International:** World Championships, Olympics, Pan Pacific, etc.

### Result Statuses

| Status | Meaning |
|--------|---------|
| DQ | Disqualified (rule infraction during race) |
| NS | No Show (entered but did not swim) |
| SCR | Scratched (withdrew before the event) |
| NT | No Time / No entry time (swimming with no seed time) |

A valid result has either a `final_time` or a status. If neither is present, the data is suspect.

## Common Commands

```bash
# Run the scraper for a specific meet
uv run python -m src.runner --source hytek_web --url "https://example.com/meet-results/"

# Run the full test suite
uv run pytest

# Run tests for a specific module
uv run pytest tests/test_scrapers.py

# Check data in the database (interactive)
uv run python -c "from src.database.client import get_client; print(get_client().table('results_full').select('*').limit(5).execute())"
```

## Data Sources

- **Hy-Tek Web Results:** Public HTML pages generated by Hy-Tek Meet Manager, the dominant meet management software in North American swimming. These pages have a consistent but sometimes varying HTML structure.
- **Swimcloud:** Historical data aggregator, useful for time histories and rankings.
- **Swim Alberta / Swimming Canada websites:** Meet schedules, qualifying times, results links.

## Environment

Supabase credentials and other configuration are managed via environment variables. See `src/config.py` for required variables.
