# Data Collection Agent

## Purpose

Discover and scrape new swim meet results into the database. This agent runs after major meet weekends to capture new results.

## Finding New Meets

### Where to Look

1. **Swim Alberta website** — Check the results/calendar section for recently completed meets. Provincial and invitational meets in Alberta will be posted here.
2. **Swimming Canada results page** — National-level meets and major interprovincial competitions.
3. **Direct Hy-Tek results URLs** — Meet hosts often publish results at URLs like `https://hostname/meets/meetname/`. Look for pages with event lists and individual results.
4. **Club websites** — Alberta clubs (e.g., Calgary Swim Club, University of Calgary Swim Club, Edmonton Keyano) post meet results on their own sites.

### Identifying Hy-Tek Result Pages

Hy-Tek web results typically have:
- An index page listing all events
- Individual event pages with tables of results
- URLs containing patterns like `/results/`, `/meets/`, or file names like `evtindex.htm`
- HTML generated by "Meet Manager" or "HY-TEK"

## Running a Scrape

```bash
uv run python -m src.runner --source hytek_web --url "https://example.com/meet-results/"
```

### Before Scraping: Check for Duplicates

Always check the `scrape_log` table before scraping a meet to avoid processing the same data twice:

```python
from src.database.client import get_client

client = get_client()
result = client.table('scrape_log').select('*').ilike('url', '%meet-name%').execute()

if result.data:
    print(f"Already scraped: {result.data}")
    # Do NOT re-scrape unless there's a specific reason (e.g., incomplete previous scrape)
```

If the meet has been scraped before, skip it unless you have reason to believe the previous scrape was incomplete.

## Expected Frequency

- **Weekly during season (September-August):** Check for new results every Monday after meet weekends.
- **Peak periods:** Provincial and national championships (typically December, March-April, July-August) — check daily for a few days after the meet.
- **Off-season (August-September):** Minimal activity, check monthly.

## Error Handling

- **Network errors:** Log the failure with the URL and timestamp. Do not retry immediately. Wait at least 1 hour before retrying.
- **Parse errors:** If the scraper fails to parse a page, log the URL and the error. This likely indicates a new Hy-Tek format variation — flag it for the improver agent.
- **Partial results:** If a scrape completes but produces fewer results than expected (e.g., a 3-day meet with only 10 results), log a warning. The meet page may still be incomplete or the format may have changed.
- **Persistent failures:** If a URL fails 3 times across different attempts, report it for manual investigation.

## Quality Checks After Scraping

After a successful scrape, verify the data is reasonable:

1. **Result count:** A typical invitational meet produces 200-2000 individual results. If you get fewer than 10, something is wrong.
2. **Time ranges:** No individual event should have a time under 10 seconds (except 50 Free SCY for seniors, which can be ~18s). No event should exceed 1800 seconds (30 minutes).
3. **Event coverage:** A multi-session meet should have results across multiple events, not just one.
4. **No empty events:** If an event page was found but no results were parsed, flag it.

```python
# Quick post-scrape validation
from src.database.client import get_client

client = get_client()
recent = client.table('results_full').select('*').eq('meet_id', MEET_ID).execute()

result_count = len(recent.data)
print(f"Results scraped: {result_count}")

if result_count < 10:
    print("WARNING: Very few results — check scrape quality")

for r in recent.data:
    if r.get('final_time') and (r['final_time'] < 10 or r['final_time'] > 1800):
        print(f"WARNING: Suspicious time {r['final_time']}s for {r.get('event_name', 'unknown event')}")
```

## Logging

After every scrape attempt (success or failure), ensure the `scrape_log` table is updated with:
- URL scraped
- Timestamp
- Status (success/failure)
- Number of results captured
- Any error messages
